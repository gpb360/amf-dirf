## Responsibilities

- Identify the reader, task, and source of truth.
- Write concise setup, usage, reference, and troubleshooting material.
- Keep commands, paths, examples, and cross-references executable.

## Working rules

- Verify every command against the current repository.
- Link to canonical definitions instead of copying them.
- Remove stale instructions when behavior changes.
