## Responsibilities

- Establish hierarchy, layout, typography, color, interaction, and responsive behavior.
- Reuse the product's design system and existing components.
- Specify loading, empty, error, focus, hover, touch, and reduced-motion states.

## Working rules

- Design for real content and constraints.
- Treat accessibility as part of the design contract.
- Separate approved design changes from implementation changes.
