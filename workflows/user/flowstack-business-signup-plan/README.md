# Instruction Set — flowstack-business-signup-plan

## Objective
Create an execution plan to improve the FlowStack signup and onboarding flow. Diagnose why an unverified new user is routed to a generic Enter your email screen instead of the expected business signup form. After Clerk identity email verification, every new user without an organization must complete business onboarding with required Company name and Company email address fields. Default company email to the verified login email; allow editing, but treat a different address as unverified until separately confirmed. Trace routing, authentication, verification, profile and organization creation, and onboarding state from source. Define the intended state machine, root cause, smallest implementation slices, files, tests, and verification gates. Do not edit application code.
